# 5.30soft_mask 구현 가이드

## 목적

TimesFM 2.5 백본 위에 올라가는 FuncDec 디코더의 시즈널 디코더를
**학습된 soft harmonic gate**로 개선한다.

기존 5.30fine_mask에서 context_span 조건(`context_span >= P/k`)으로 하드하게 0을
박던 방식을 **backbone embedding에서 예측한 per-harmonic gate**로 대체한다.
물리적으로 불가능한 주파수(`fd >= P/k`)는 여전히 basis에서 0으로 처리한다.

소프트 마스크 이외의 모든 사항(학습 순서, 데이터, 캐싱, 평가, 모델 구조)은
**5.30fine_mask와 동일**하게 유지한다.

---

## 아키텍처 변경점 (fine_mask → soft_mask)

### 1. Basis 빌드 규칙 변경

| | fine_mask | soft_mask |
|---|---|---|
| 물리 조건 `fd < P/k` | 적용 (basis 0) | 동일하게 적용 |
| context span 조건 `context_span >= P/k` | 적용 (basis 0) | **제거 — gate로 대체** |
| basis 소스 | 캐시 파일 (`fourier_basis_fine_mask_*.pt`) | **on-the-fly 계산** (freq + horizon만 필요) |

```
build_soft_mask_basis(freq, horizon):
    for each family, period P, harmonic k:
        if fd < P/k:              # 물리 조건만
            basis[k] = sin/cos(...)
        else:
            basis[k] = 0          # 물리적으로 불가능 → 여전히 0
```

**중요**: context_len은 이 함수에서 사용하지 않는다.
backbone_emb 캐시 파일은 그대로 재사용 (변경 없음).

### 2. SeasonalDecoder 변경

```
# fine_mask SeasonalDecoder
forward(emb, d_basis, w_basis, m_basis, y_basis):
    coef_d, coef_w, coef_m, coef_y = MLP(emb) → head
    s_d = bmm(d_basis, coef_d)
    seasonal = s_d + s_w + s_m + s_y

# soft_mask SeasonalDecoder (NEW)
forward(emb, d_basis, w_basis, m_basis, y_basis):
    coef_d, coef_w, coef_m, coef_y = MLP(emb) → head   # 동일
    gates = sigmoid(GatingMLP(emb))                      # NEW: (B, 24)
    #   daily:   gates[:,  0:10]   → (B, 10)
    #   weekly:  gates[:, 10:14]   → (B, 4)
    #   monthly: gates[:, 14:16]   → (B, 2)
    #   yearly:  gates[:, 16:24]   → (B, 8)
    coef_d *= repeat_interleave(gate_d, 2, dim=1)        # (B,20)*(B,20)
    s_d = bmm(d_basis, coef_d)
    seasonal = s_d + s_w + s_m + s_y
```

#### GatingMLP 구조
```python
GatingMLP: embed_dim(1280) → Linear → ReLU → Linear → 24
# 마지막 Linear의 weight/bias를 0으로 초기화
# → 초기 gate = sigmoid(0) = 0.5 (중립)
```

게이트 총 수: K_MAX 합 = 10 + 4 + 2 + 8 = **24**

### 3. decomp_funcdec 변경

`forward()` 반환 dict에 `"gates"` 키 추가:
```python
return pred_n, {
    "trend": ..., "seasonal": ..., "residual": ...,
    "delta": ..., "seasonal_coefficients": ...,
    "gates": gates,   # (B, 24) — train.py에서 L1 loss 계산에 사용
}
```

### 4. Loss 변경

```python
total_loss = reconstruction_loss + gate_l1_weight * gates.mean()
# gates: post-sigmoid 값 (0~1), L1은 abs()와 동치
# gate_l1_weight: 하이퍼파라미터 (권장 범위: 1e-4 ~ 1e-2)
# L1은 모든 학습 phase에서 동일하게 적용
```

## 점검 체크리스트

다른 CLI가 구현 후 반드시 확인해야 할 사항들.

### [A] build_soft_mask_basis

- [ ] `context_span` 변수를 사용하지 않는다
- [ ] `context_len` 인자를 받지 않는다 (시그니처: `freq, horizon`)
- [ ] `fd < P/k` 조건만으로 basis 0 여부 결정
- [ ] `fd >= P/k` 인 harmonic은 여전히 0 (물리 불가 조건 유지)
- [ ] fine_mask basis보다 non-zero 컬럼이 같거나 더 많다
  - 예: daily data (fd=1.0)에서 weekly (P=7d, k=1: 7d harmonic) → fine_mask에서는
    context_span=512d >= 7d이므로 동일하게 켜짐
  - 예: hourly data (fd=1/24)에서 monthly k=1 (30.4d harmonic), context_span=512/24=21.3d < 30.4d
    → fine_mask: 꺼짐 / soft_mask: **켜짐** (gate로 학습)

### [B] HarmonicGatingNetwork

- [ ] 출력 차원 = 24 (K_MAX 합계)
- [ ] 마지막 Linear의 weight=0, bias=0으로 초기화
- [ ] `sigmoid()` 적용 후 반환 → 값 범위 [0,1]
- [ ] forward 입력: emb만 (basis 입력 없음)

### [C] SeasonalDecoder.forward

- [ ] gate 슬라이싱 인덱스 정확성:
  ```
  gate_daily   = gates[:,  0:10]   # K_MAX["daily"] = 10
  gate_weekly  = gates[:, 10:14]   # K_MAX["weekly"] = 4
  gate_monthly = gates[:, 14:16]   # K_MAX["monthly"] = 2
  gate_yearly  = gates[:, 16:24]   # K_MAX["yearly"] = 8
  ```
- [ ] `repeat_interleave(gate, 2, dim=1)` 후 coef와 곱셈 — bmm 이전에 적용
- [ ] basis 형상과 gated_coef 형상이 bmm에서 일치
  - basis: `(B, H, 2*K_MAX[f])`, coef_gated: `(B, 2*K_MAX[f])` → unsqueeze(-1) 후 bmm
- [ ] `return_coefficients=True` 시 반환되는 coefficients는 **gate 적용 전** 값
  (gate 정보는 별도로 전달하거나 gates를 함께 반환)

### [D] FuncDecModel.forward

- [ ] 반환 dict에 `"gates"` 키 존재 → `(B, 24)` tensor
- [ ] forward 시그니처는 fine_mask와 동일: `(emb, daily, weekly, monthly, yearly)`

## 주요 의도 정리

**왜 soft mask인가?**
- fine_mask의 context_span 조건은 heuristic: 컨텍스트가 충분히 길어야만 해당 주기를 켠다
- 실제로는 짧은 컨텍스트에도 해당 주기의 단서가 backbone embedding에 담길 수 있다
- embedding이 "이 데이터는 월간 패턴이 있다"는 정보를 갖고 있다면
  직접 gate를 올려서 monthly harmonic을 활성화할 수 있어야 한다

**왜 L1 regularization인가?**
- gate를 자유롭게 두면 모든 harmonic을 약하게 켜는 방향으로 수렴할 가능성
- L1은 gate를 0으로 끌어당겨 불필요한 harmonic을 실제로 끄도록 유도
- 결과적으로 fine_mask의 hard sparsity를 soft하게 학습하는 효과

**캐싱 전략 근거:**
- basis는 freq + horizon의 함수 (context_len 불필요) → on-the-fly가 zero-cost에 가까움
- backbone_emb 캐시는 여전히 가장 비용이 큰 부분 → 그대로 재사용
- fourier_basis 캐시 파일은 context_len 정보가 내포되어 soft_mask와 불일치 → 무시
